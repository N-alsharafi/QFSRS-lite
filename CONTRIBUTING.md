# Contributing

Suggest features or report bugs via [GitHub Issues](https://github.com/N-alsharafi/QFSRS-lite/issues).

## Pull Request Rules

### No blind AI code

You may use AI assistants or coding agents, but you **must understand all the code** before submitting. In your pull request, explain what each change does and why. If you cannot explain it, do not submit it.

### Single-feature, high quality

- One feature or fix per pull request
- PRs that bundle multiple unrelated changes will not be merged
- Code must be clean, tested where appropriate, and follow existing patterns

### Tests:

- The main operational logic is contained in `.ts` files, if your pull request has a ts file (i.e adds or updates operational logic), you must submit unit tests along with it that, and explain the logic and what the unit tests test.
